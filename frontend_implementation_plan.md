# План реализации Angular Frontend для Grocery Store

На основе спроектированного NestJS API, клиентское приложение на Angular будет представлять собой систему управления магазином, разделенную на два основных рабочих пространства: интерфейс кассира (Point of Sale) и панель администратора (управление каталогом, запасами и скидками).

## 1. Архитектура приложения и маршрутизация

Приложение будет использовать современный подход Angular со Standalone-компонентами и реактивным управлением состоянием через Signals.

### Основные маршруты (Routes):
*   **`/pos`** (Point of Sale) — Рабочее место кассира (оформление заказа, добавление товаров, оплата).
*   **`/admin/catalog`** — Управление каталогом товаров (CRUD).
*   **`/admin/inventory`** — Управление остатками на складе.
*   **`/admin/discounts`** — Настройка и просмотр скидочных кампаний.

---

## 2. Структура слоев (Services & State)

Взаимодействие с NestJS API будет инкапсулировано в сервисах. Мы будем использовать Angular `HttpClient` для запросов.

1.  **`CatalogService`**
    *   Методы: `getItems()`, `getItemByBarcode(barcode)`, `createItem(item)`, `deleteItem(barcode)`.
2.  **`InventoryService`**
    *   Методы: `updateStock(barcode, count)`, `getStock(barcode)`.
3.  **`DiscountService`**
    *   Методы: `getCampaigns()`, `createCampaign(campaign)`. Для создания потребуется UI-конструктор JSON-конфигураций (выбор стратегии и критериев).
4.  **`CheckoutService` (Stateful)**
    *   Управляет процессом покупки. Содержит Signals для текущего `orderId` и состояния корзины (список добавленных товаров, итоговая сумма).
    *   Методы: `startOrder()`, `addItem(barcode, quantity)`, `getCartState()`, `pay(amount)`.
    *   *Примечание:* Добавление товара вызывает API (`POST /api/checkout/orders/:orderId/items`), после чего обновляет локальный Signal результатами от сервера (чтобы сервер сам рассчитал скидки).

## 3. Пользовательские интерфейсы (Components)

### 3.1. POS Интерфейс (Point of Sale) — `PosComponent`
Интерфейс кассира, состоящий из двух блоков:
*   **Панель ввода:**
    *   Поле ввода штрихкода (имитация сканера).
    *   Кнопка "Добавить товар" и выбор количества.
    *   Список быстрого доступа (часто покупаемые товары из каталога).
*   **Чек (Корзина):**
    *   Список добавленных позиций (название, количество, цена до скидки, цена после скидки).
    *   Отображение `Subtotal` и `Total` (берутся из API).
    *   Кнопка "Оплатить", открывающая модальное окно ввода суммы (`paymentAmount`) для расчета сдачи.
    *   Вывод итогового чека (`Receipt`).

### 3.2. Панель Администратора (Admin Dashboard)
*   **`AdminCatalogComponent`:** Таблица всех товаров. Форма для создания нового товара (ввод `barcode`, `name`, `category`, `price`).
*   **`AdminInventoryComponent`:** Таблица остатков. Возможность быстро скорректировать количество (инкремент/декремент).
*   **`AdminDiscountsComponent`:**
    *   Отображение текущих акций.
    *   **Builder (Конструктор акций):** Форма, позволяющая выбрать тип критерия (Категория/Товар/Композитный) и тип стратегии (Процент/Сумма/Декоратор), которая затем сериализуется в JSON и отправляется на сервер.

## 4. UI/UX Дизайн (Syntactic Dark)

Все компоненты будут стилизованы в едином премиальном дизайне "Syntactic Dark" с элементами glassmorphism:
*   **Палитра:** Темные насыщенные тона (HSL), современная типографика (Inter/Roboto).
*   **Стилизация:** Плавные градиенты, эффекты стекла (blur) для карточек товаров и модальных окон оплаты.
*   **Анимации:** Микро-взаимодействия при добавлении товара в корзину (появление строки) и при применении скидки (подсветка измененной цены).

---

## 5. Пошаговый план разработки

**Шаг 1: Базовая структура и Роутинг**
1.  Настройка Angular Router для разделения `/pos` и `/admin/*`.
2.  Создание основного Layout с навигационной панелью (Header) для переключения между режимом кассира и админкой.

**Шаг 2: Интеграция с API (Сервисы)**
1.  Настройка `HttpClient` (в `app.config.ts`).
2.  Создание `CatalogService`, `InventoryService`, `DiscountService` и `CheckoutService` с интерфейсами TypeScript, дублирующими DTO бэкенда.

**Шаг 3: Разработка Admin Dashboard**
1.  Верстка `AdminCatalogComponent` (таблица + форма). Подключение к API.
2.  Верстка `AdminInventoryComponent` (управление запасами).
3.  Создание базового `AdminDiscountsComponent`.

**Шаг 4: Разработка POS (Интерфейс кассира)**
1.  Верстка экрана `PosComponent` с использованием Signals для реактивного отображения корзины.
2.  Интеграция ввода штрихкодов с API.
3.  Модальное окно оплаты и отображение чека (Receipt).

**Шаг 5: Стилизация и Полировка (Glassmorphism & UX)**
1.  Внедрение глобальных CSS-переменных (дизайн-системы).
2.  Добавление микро-анимаций для списка корзины и интерактивных элементов.
